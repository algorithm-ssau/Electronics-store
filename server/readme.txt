Серверная часть для обеспечения работы приложения онлайн магазина предоставляет следующие сущности:
Customer, Order, Product, Template
У каждой сущности в приложении есть свой уникальный id.
Customer это объект покупателя в магазине. У него есть имя, уникальный логин и пароль, количество денег
на счету и список id его заказов (Order).
Order это заказ пользователя, он создается, при покупки пользователем каких либо товаров(Product)
и содержит в себе статус заказа и id продуктов, которые заказал пользователь
Product это экземпляр товара созданного по Template, который заказал пользователь. У него есть
id заказа и id темплейта по которому он был создан, а также цена, имя и описание
Template это шаблон для продукта. Когда пользователь заходит в каталог товаров перед ним
появляется список с описанием всех возможных продуктов - это и есть темплейты. У темплейта
есть имя цена и описание

Подготовка к запуску PythonAPI:
    1) Запустить в терминале команду python3 -m venv env
    2) Активировать виртуальную среду разработки вводом в терминал команды source env/bin/activate
    3) Запустить в терминале команду pip3 install -r pip_packages.txt
Запуск Python API:
    1) Запустить в терминале команду uvicorn main:app
    2) Поздравляю, простейшее API на языке Python было успешно запущено
Функционал Python API:
    Основная задача данного API - возвращать общедоступную служебную информацию, хранящуюся в файле .env
    На данный момент можно получить в JSON формате следующие данные:
        1) Версию Сервера (/api/appinfo/server/version)
        2) Версию Клиента (/api/appinfo/client/version)
        3) Дату последнего обновления Сервера (/api/appinfo/server/last_update)
        4) Дату последнего обновления Клиента (/api/appinfo/client/last_update)
        5) Список разработчиков с указанием позиций в команде (/api/appinfo/developers)
